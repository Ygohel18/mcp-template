# MCP Connection Guide 🔌

This guide provides specific configuration examples for connecting the `@ygohel18/mcp-template` server to various platforms.

---

## 1. 📂 Local Stdio (Desktop Clients)

Best for **Claude Desktop**, **Cline**, and **Roo-Code**.

### Claude Desktop
Add this to your `claude_desktop_config.json`:
```json
{
  "mcpServers": {
    "ygohel-template": {
      "command": "node",
      "args": ["/PATH/TO/mcp-template/dist/index.js"],
      "env": {
        "MCP_TRANSPORT": "stdio"
      }
    }
  }
}
```

---

## 2. 🌐 SSE (Remote & Studio Clients)

Best for **LM Studio**, **VS Code Extensions**, and **.codex**.

### LM Studio
1. Start the server: `npm run dev` (Ensure `MCP_TRANSPORT=sse` in `.env`).
2. In LM Studio, go to the **MCP Section**.
3. Add a new SSE server with the URL: 
   `http://localhost:3000/sse` OR `http://localhost:3000/mcp`

### VS Code (.codex / MCP Config)
Point your extension to:
`http://localhost:3000/sse`

---

## 3. 🤖 ChatGPT (OAuth Connector)

To use your tools within ChatGPT:
1. **Host your server** (use Ngrok if testing locally: `ngrok http 3000`).
2. Create a **Custom GPT** -> **Configure** -> **Actions** -> **Create New Action**.
3. **Authentication**: 
   - Type: `OAuth`
   - Client ID: (From your `.env`)
   - Client Secret: (From your `.env`)
   - Authorization URL: `https://your-ngrok-url.app/oauth/authorize`
   - Token URL: `https://your-ngrok-url.app/oauth/token`
4. **Import from URL**: `https://your-ngrok-url.app/.well-known/ai-plugin.json` (or paste the OpenAPI spec generated by MCP).

---

## 4. 🌩️ Claude (Kiro Powers / Bedrock)

This template allows Claude to use these tools via AWS Bedrock:
1. Set `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY` in `.env`.
2. Claude will be able to call `aws_bedrock_generate` tool to perform high-level reasoning tasks.
